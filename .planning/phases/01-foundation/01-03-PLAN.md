<task type="auto">
  <name>Task 2: Create Proxy-based reactive state store, SSE listener, tab router, and task list view</name>
  <files>public/app.js</files>
  <action>
  Create `public/app.js` with the complete frontend foundation:

  1. **HTML escape utility** (prevent XSS from task/agent data -- all dynamic strings MUST go through this before insertion):
     - Function `esc(str)` replaces &, <, >, " with HTML entities. Returns empty string for null/undefined.

  2. **Reactive state store** using Proxy:
     - Internal `_state` object: { tasks: [], activity: [], notifications: [], activeTab: 'board', sseConnected: false }
     - Internal `_listeners` object: maps property names to arrays of callback functions
     - `state` = new Proxy over `_state`. The `set` trap: stores the new value, then calls all registered listeners for that property with (newValue, oldValue).
     - `onChange(prop, fn)` function: registers a listener for a given state property.
     - This gives one-way data flow: update `state.tasks = newArray` and all 'tasks' listeners fire automatically.

  3. **Fetch helpers**:
     - `async function api(path, options)`: prepends '/api' to path, sets Content-Type application/json, JSON.stringifies body if present, returns parsed JSON response. On non-2xx, throws with status and response body.
     - `async function loadTasks()`: calls api('/tasks'), sets `state.tasks = tasks`
     - `async function loadActivity()`: calls api('/activity?limit=50'), sets `state.activity = activity`

  4. **SSE connection**:
     - `function connectSSE()`: creates `new EventSource('/api/events')`
     - onopen: set `state.sseConnected = true`
     - onerror: set `state.sseConnected = false`
     - Listen for 'task:created': parse JSON data, append to state.tasks via spread (new array, not mutation)
     - Listen for 'task:updated': parse data, map over state.tasks replacing matching id (new array)
     - Listen for 'task:deleted': parse data, filter out matching id from state.tasks (new array)
     - Listen for 'activity:new': parse data, prepend to state.activity, slice to 100 max

  5. **Tab router** (hash-based):
     - `tabs` object mapping hash strings to { render, label } objects: #/board, #/agents, #/activity, #/inbox
     - `function navigate()`: reads location.hash (default #/board), looks up tab, calls tab.render(mainElement), updates nav-tab active classes
     - Wire to hashchange and DOMContentLoaded events

  6. **Render functions** (component pattern -- functions that write to a container element):

     **renderBoard(container)**: Groups state.tasks by status (todo, in_progress, review, done). Renders a 4-column CSS grid. Each column has a header with status label and count, then task cards below. Each task card (built via `taskCard(task)` helper) shows: title (escaped via esc()), priority badge with appropriate color class, agent badge if assigned, relative time. Include a "New Task" button at the top that calls showCreateTaskForm(). Use event delegation on the container -- a single click listener that checks `e.target.closest('[data-action]')` and dispatches to action handlers.

     **renderAgents(container)**: Renders a grid of 8 agent cards. Hardcode the agent list: [{id:'jarvis', name:'Jarvis', role:'Main orchestrator'}, {id:'scout', name:'Scout', role:'Research & briefs'}, {id:'analyst', name:'Analyst', role:'Data analysis'}, {id:'forge', name:'Forge', role:'Builder & coder'}, {id:'sentinel', name:'Sentinel', role:'Security audits'}, {id:'broker', name:'Broker', role:'Portfolio & finance'}, {id:'ops', name:'Ops', role:'Infrastructure'}, {id:'hunter', name:'Hunter', role:'Job hunting'}]. Each card: name, role, "idle" badge. Uses card CSS class.

     **renderActivityView(container)**: Renders state.activity as a list. Each entry: formatted timestamp, event_type as a badge, summary text (escaped). Shows "No activity yet" if empty.

     **renderInbox(container)**: Placeholder showing "No notifications" with muted text.

     **showCreateTaskForm()**: Appends an overlay div to document.body. The overlay contains a form with: title input (required), description textarea, priority select (critical/high/medium/low, default medium), agent_id select (unassigned + 8 agents). Submit button calls api('/tasks', { method: 'POST', body: formData }). On success: remove the overlay. Do NOT manually update state.tasks -- the SSE 'task:created' event handles that automatically, proving one-way data flow. On error: show error text in the form. Cancel button or Escape or clicking overlay background removes it.

     **taskCard(task)** helper: Returns an HTML string for a single task card. All dynamic text goes through esc(). Priority determines a CSS class for color. Agent badge uses agent color var if assigned. Shows relative time via a `timeAgo(dateStr)` helper.

     **timeAgo(dateStr)** helper: Converts ISO date string to relative time ("2m ago", "1h ago", "3d ago"). Simple implementation using Date.now() - Date.parse().

  7. **Keyboard shortcuts**:
     - Listen for keydown on document
     - Skip if target is INPUT, TEXTAREA, or SELECT (user is typing)
     - Skip if meta/ctrl/alt held (allow browser shortcuts)
     - Keys 1-4 map to tab hashes: 1=#/board, 2=#/agents, 3=#/activity, 4=#/inbox
     - Escape: remove any element with class 'overlay'

  8. **SSE status indicator**:
     - onChange('sseConnected', ...) updates the #sse-status element: "LIVE" badge-green when true, "OFFLINE" badge-red when false

  9. **State change listeners for re-rendering**:
     - onChange('tasks', ...) re-renders board view if activeTab is 'board'
     - onChange('activity', ...) re-renders activity view if activeTab is 'activity'

  10. **Initialization (DOMContentLoaded)**:
      - Load tasks and activity in parallel via Promise.all
      - Connect SSE
      - Call navigate() for initial route
      - Wire hashchange listener
      - Set status bar right side to current SAST time, update every 60 seconds

  CRITICAL RULES:
  - NEVER read state from the DOM. Always read from `state.tasks`, `state.activity`, etc.
  - All dynamic user/agent content MUST go through `esc()` before being placed in HTML strings. This includes task titles, descriptions, agent names from API, activity summaries.
  - State updates always create new arrays (spread/map/filter), never mutate in place, so the Proxy set trap fires.
  - Use textContent where possible for simple text updates (like status bar). Use container writes with escaped HTML strings for complex components.
