<task type="auto">
  <name>Task 1: Create SSE broker module with event bus and broadcast</name>
  <files>sse.js</files>
  <action>
  Create `sse.js` at project root:

  1. `const EventEmitter = require('node:events');`
  2. Create and export `bus = new EventEmitter()` -- the central event bus all mutations emit to.
  3. `const clients = new Set();` -- holds active SSE response objects.
  4. `let eventId = 0;` -- monotonic counter for SSE event IDs (in-memory; the activity_log table provides persistence).

  5. `function handleSSE(req, res)`:
     - Set headers: Content-Type text/event-stream, Cache-Control no-cache, Connection keep-alive, X-Accel-Buffering no
     - Write initial comment `:ok\n\n` to flush the connection
     - Send `retry: 3000\n\n` to set client reconnect interval
     - Add `res` to `clients` Set
     - Set up keepalive: `const keepAlive = setInterval(() => res.write(': keepalive\n\n'), 30000);`
     - Clean up on close: `req.on('close', () => { clients.delete(res); clearInterval(keepAlive); });`

  6. `function broadcast(eventType, data)`:
     - Increment `eventId`
     - Format SSE message: `id: ${eventId}\nevent: ${eventType}\ndata: ${JSON.stringify(data)}\n\n`
     - Iterate `clients` Set and write the message to each. Wrap each write in try/catch -- if write fails, delete that client from the Set (dead connection cleanup).

  7. Wire bus events to broadcast:
     - `bus.on('task:created', (d) => broadcast('task:created', d));`
     - `bus.on('task:updated', (d) => broadcast('task:updated', d));`
     - `bus.on('task:deleted', (d) => broadcast('task:deleted', d));`
     - `bus.on('activity:new', (d) => broadcast('activity:new', d));`

  8. Export: `module.exports = { bus, handleSSE, broadcast };`
  </action>

<task type="auto">
  <name>Task 2: Replace placeholder API routes with real CRUD + SSE wiring in server.js</name>
  <files>server.js</files>
  <action>
  Modify `server.js` (created in Plan 01) to replace all placeholder routes with real implementations:

  1. Add require at top: `const { bus, handleSSE } = require('./sse');`

  2. Replace route handlers:

  **GET /api/tasks**:
  - Call `stmts.getAllTasks.all()`
  - Return `res.json({ tasks })`

  **POST /api/tasks**:
  - Read body via `readBody(req)`
  - Validate: title is required (non-empty string). Return 400 if missing.
  - Set defaults: status='todo', priority='medium', sort_order=0
  - Call `stmts.insertTask.run({ title, description: body.description || null, status: body.status || 'todo', priority: body.priority || 'medium', agent_id: body.agent_id || null, project_id: body.project_id || null, sort_order: body.sort_order || 0 })`
  - Get the created task: `const task = stmts.getTaskById.get(result.lastInsertRowid);`
  - Log activity: `stmts.insertActivity.run({ event_type: 'task.created', agent_id: null, task_id: task.id, project_id: task.project_id, summary: 'Task created: ' + task.title, detail_json: JSON.stringify(task) })`
  - Emit: `bus.emit('task:created', task)`
  - Emit: `bus.emit('activity:new', { event_type: 'task.created', summary: 'Task created: ' + task.title, task_id: task.id })`
  - Return `res.json({ task }, 201)`

  **PATCH /api/tasks/:id**:
  - Extract id from URL path segment
  - Read body
  - Get existing task: `stmts.getTaskById.get(id)`. Return 404 if not found.
  - Merge fields: only update fields present in body, keep existing values for others
  - Call `stmts.updateTask.run({ id, title, description, status, priority, agent_id, sort_order })`
  - Get updated task: `stmts.getTaskById.get(id)`
  - Log activity with appropriate event_type (use 'task.updated', or 'task.moved' if status changed)
  - Emit `bus.emit('task:updated', updatedTask)` and `bus.emit('activity:new', ...)`
  - Return `res.json({ task: updatedTask })`

  **DELETE /api/tasks/:id**:
  - Extract id, verify task exists (404 if not)
  - Get task before delete for activity log
  - Call `stmts.deleteTask.run(id)`
  - Log activity: 'task.deleted'
  - Emit `bus.emit('task:deleted', { id })` and `bus.emit('activity:new', ...)`
  - Return `res.json({ ok: true })`

  **GET /api/events**:
  - Call `handleSSE(req, res)` from sse.js

  **GET /api/activity**:
  - Parse query param `limit` (default 50, max 200)
  - Call `stmts.getRecentActivity.all(limit)`
  - Return `res.json({ activity })`

  **GET /api/notifications**:
  - Keep as placeholder for now (returns empty array) -- notifications are consumed in Phase 4
  - Return `res.json({ notifications: [] })`

  3. Make sure the try/catch wrapper from Plan 01 covers these new implementations. Log errors to console.error with the full error object.
