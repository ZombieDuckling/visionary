<task type="auto">
  <name>Task 1: Create package.json, db.js with migration system, and initialize SQLite</name>
  <files>package.json, db.js</files>
  <action>
  1. Create `package.json` at project root:
     - name: "visionary"
     - version: "0.1.0"
     - main: "server.js"
     - scripts: { "start": "node server.js", "dev": "node --watch server.js" }
     - dependencies: { "better-sqlite3": "^11.0.0" }

  2. Create `db.js` that exports `db` and `stmts`:
     - `const Database = require('better-sqlite3');`
     - `const db = new Database('./visionary.sqlite');`
     - Apply PRAGMAs immediately after opening (order matters -- WAL first):
       ```
       db.pragma('journal_mode = WAL');
       db.pragma('synchronous = NORMAL');
       db.pragma('foreign_keys = ON');
       db.pragma('busy_timeout = 5000');
       db.pragma('cache_size = -20000');
       db.pragma('temp_store = MEMORY');
       ```
     - Create migration system:
       - `db.exec('CREATE TABLE IF NOT EXISTS schema_version (version INTEGER NOT NULL DEFAULT 0)')`
       - `db.prepare('INSERT OR IGNORE INTO schema_version (rowid, version) VALUES (1, 0)').run()`
       - Define a `migrations` array where each element is a SQL string. Index 0 = migration to version 1, etc.
       - Migration runner: get current version, run all migrations above it inside a transaction, update version after each.
     - Migration 1 (version 0 -> 1): Create all 5 tables with CHECK constraints and indexes. Use the schema from ARCHITECTURE.md:
       - `projects` (id INTEGER PK AUTOINCREMENT, name TEXT NOT NULL, slug TEXT UNIQUE NOT NULL, description TEXT, color TEXT DEFAULT '#00ff41', status TEXT DEFAULT 'active' CHECK(status IN ('active','paused','archived')), created_at TEXT DEFAULT (datetime('now')), updated_at TEXT DEFAULT (datetime('now')))
       - `tasks` (id INTEGER PK AUTOINCREMENT, project_id INTEGER REFERENCES projects(id), title TEXT NOT NULL, description TEXT, status TEXT DEFAULT 'todo' CHECK(status IN ('todo','in_progress','review','done')), priority TEXT DEFAULT 'medium' CHECK(priority IN ('critical','high','medium','low')), agent_id TEXT, created_at TEXT DEFAULT (datetime('now')), updated_at TEXT DEFAULT (datetime('now')), completed_at TEXT, sort_order INTEGER DEFAULT 0)
       - `agent_runs` (id INTEGER PK AUTOINCREMENT, task_id INTEGER REFERENCES tasks(id), agent_id TEXT NOT NULL, session_id TEXT, message TEXT NOT NULL, status TEXT DEFAULT 'pending' CHECK(status IN ('pending','running','completed','failed','timeout')), result_json TEXT, result_text TEXT, error TEXT, delivery_status TEXT, duration_ms INTEGER, started_at TEXT, completed_at TEXT, created_at TEXT DEFAULT (datetime('now')))
       - `notifications` (id INTEGER PK AUTOINCREMENT, agent_run_id INTEGER REFERENCES agent_runs(id), type TEXT NOT NULL CHECK(type IN ('info','action','warning','error')), title TEXT NOT NULL, body TEXT, action_type TEXT, action_data TEXT, read INTEGER DEFAULT 0, dismissed INTEGER DEFAULT 0, created_at TEXT DEFAULT (datetime('now')))
       - `activity_log` (id INTEGER PK AUTOINCREMENT, event_type TEXT NOT NULL, agent_id TEXT, task_id INTEGER, project_id INTEGER, summary TEXT NOT NULL, detail_json TEXT, created_at TEXT DEFAULT (datetime('now')))
       - Indexes: idx_tasks_status, idx_tasks_project, idx_agent_runs_task, idx_agent_runs_status, idx_notifications_read, idx_activity_created (DESC), idx_activity_agent
     - Export prepared statements object `stmts` with these commonly-used queries:
       - `insertTask`: INSERT INTO tasks with named parameters (@title, @description, @status, @priority, @agent_id, @project_id, @sort_order)
       - `getTasksByStatus`: SELECT t.*, p.name as project_name, p.color as project_color FROM tasks t LEFT JOIN projects p ON t.project_id = p.id WHERE t.status = ? ORDER BY t.sort_order, t.created_at DESC
       - `getTaskById`: SELECT t.*, p.name as project_name, p.color as project_color FROM tasks t LEFT JOIN projects p ON t.project_id = p.id WHERE t.id = ?
       - `updateTask`: UPDATE tasks SET title=@title, description=@description, status=@status, priority=@priority, agent_id=@agent_id, sort_order=@sort_order, updated_at=datetime('now'), completed_at=CASE WHEN @status='done' THEN datetime('now') ELSE completed_at END WHERE id=@id
       - `deleteTask`: DELETE FROM tasks WHERE id = ?
       - `getAllTasks`: SELECT t.*, p.name as project_name, p.color as project_color FROM tasks t LEFT JOIN projects p ON t.project_id = p.id ORDER BY t.sort_order, t.created_at DESC
       - `insertActivity`: INSERT INTO activity_log (event_type, agent_id, task_id, project_id, summary, detail_json) VALUES (@event_type, @agent_id, @task_id, @project_id, @summary, @detail_json)
       - `getRecentActivity`: SELECT * FROM activity_log ORDER BY created_at DESC LIMIT ?
     - `module.exports = { db, stmts };`

  IMPORTANT: Use `db.exec()` with the full SQL block as a single string for DDL in migrations. Wrap the migration runner in `db.transaction()`.
  </action>

<task type="auto">
  <name>Task 2: Create HTTP server with static file serving and router skeleton</name>
  <files>server.js, public/index.html</files>
  <action>
  1. Create `public/` directory and `public/index.html`:
     - Minimal valid HTML5 document
     - Title: "Visionary Mission Control"
     - Meta charset UTF-8, viewport meta tag
     - Link to `styles.css` (will be created in Plan 03)
     - Script tag loading `app.js` with defer (will be created in Plan 03)
     - Body contains a div#app with a simple "Visionary Mission Control - Loading..." placeholder
     - Dark background inline style on body as fallback: `background: #0a0e14; color: #e0e0e0; font-family: monospace;`

  2. Create `server.js`:
     - `const http = require('node:http');`
     - `const fs = require('node:fs');`
     - `const path = require('node:path');`
     - `const { URL } = require('node:url');`
     - `const { db, stmts } = require('./db');`
     - Read HTML file once at startup: `const indexHTML = fs.readFileSync(path.join(__dirname, 'public', 'index.html'), 'utf8');`
     - MIME type map: `{ '.html': 'text/html', '.css': 'text/css', '.js': 'text/javascript', '.json': 'application/json' }`
     - Helper: `readBody(req)` returns a Promise that collects request body chunks and JSON.parses them. Handle parse errors by returning null.
     - Static file server function: for paths not starting with `/api/`, try to serve from `public/` directory. Use `path.join(__dirname, 'public', pathname)` with a guard to prevent directory traversal (ensure resolved path starts with the public dir). If file not found, serve indexHTML (SPA fallback).
     - Router: `http.createServer(async (req, res) => { ... })` with:
       - Attach `res.json = (data, status = 200) => { res.writeHead(status, { 'Content-Type': 'application/json' }); res.end(JSON.stringify(data)); };`
       - Parse URL: `const url = new URL(req.url, 'http://localhost');`
       - Route table as if/else chain:
         - `GET /` -> serve indexHTML with Content-Type text/html
         - `GET /api/tasks` -> placeholder: `res.json({ tasks: [] })`
         - `POST /api/tasks` -> placeholder: `res.json({ ok: true }, 201)`
         - `PATCH /api/tasks/:id` -> placeholder: `res.json({ ok: true })`
         - `DELETE /api/tasks/:id` -> placeholder: `res.json({ ok: true })`
         - `GET /api/events` -> placeholder comment: SSE endpoint (Plan 02 will implement)
         - `GET /api/activity` -> placeholder: `res.json({ activity: [] })`
         - `GET /api/notifications` -> placeholder: `res.json({ notifications: [] })`
         - Default for /api/* -> 404 JSON: `res.json({ error: 'Not found' }, 404)`
         - Default for everything else -> static file serve
       - Extract path params with `url.pathname.split('/')[3]` for :id routes
       - Wrap the entire handler body in try/catch, return 500 JSON on error
     - Listen on `127.0.0.1:3333` with startup log: `console.log('Visionary Mission Control running at http://127.0.0.1:3333')`
     - Graceful shutdown: `process.on('SIGINT', () => { db.close(); server.close(); process.exit(0); });` and same for SIGTERM.

  IMPORTANT: Do NOT embed HTML in a template literal in server.js. The HTML lives in public/index.html and is read via fs.readFileSync. This is the number one pitfall from research -- template literal escaping causes XSS and debugging nightmares.
  </action>
