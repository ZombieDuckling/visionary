**Server-side: GET /api/agents endpoint** in server.js:

1. Add new prepared statements in db.js (add to the stmts object):

   - `getLatestRunPerAgent`: SELECT agent_id, status, completed_at, started_at, duration_ms, result_text FROM agent_runs WHERE id IN (SELECT MAX(id) FROM agent_runs GROUP BY agent_id)
   - `getRunningAgents`: SELECT agent_id FROM agent_runs WHERE status = 'running'

2. Add GET /api/agents route in server.js (inside the /api/ block, before the 404 fallback). The route should:
   - Define an array of 8 agent config objects with id, name, role, and model fields. Use model value 'claude-sonnet-4-20250514' for all agents.
   - Query latestRuns and runningAgents from the new prepared statements
   - Build a map of agent_id -> latest run row
   - For each agent config, determine status: 'active' if agent_id is in runningAgents, 'error' if latest run status is 'failed', otherwise 'idle'
   - Return each agent with: id, name, role, model, status, last_activity (completed_at or started_at from latest run, null if none), last_run_status, last_run_duration_ms, last_run_summary (first 120 chars of result_text)

**Command Bar (Cmd+K)** in public/app.js:

1. Create showCommandBar() function:
   - Creates an overlay element with classes "overlay command-bar-overlay" (having "overlay" class means existing Escape handler will close it)
   - Overlay positioned near top of viewport (not centered like task form)
   - Inner structure: a div with class "command-bar" containing an input with class "command-input" and a div with class "command-hint"
   - Input placeholder: "Type a command... (@agent message, /board, /agents)"
   - Hint text below input using kbd elements: "@ assign agent / navigate Enter create task Esc close"
   - Auto-focus the input after appending to DOM
   - Close on clicking the overlay background (outside the command-bar div) -- use same pattern as showCreateTaskForm: overlay click handler checks e.target === overlay

2. Create hideCommandBar() -- finds and removes .command-bar-overlay element if present.

3. Create toggleCommandBar() -- if .command-bar-overlay exists in DOM, remove it; otherwise call showCommandBar().

4. Wire command bar input events (inside showCommandBar, after creating elements):
   - Listen for keydown on the command input element
   - If key is Enter:
     - Get trimmed value from input
     - If empty, close and return
     - If starts with "/": map known routes (/board, /agents, /activity, /inbox) to hash navigation. Set location.hash. Close bar.
     - If starts with "@": use regex to extract agent id and message text. Create task via api('/tasks', { method: 'POST', body: { title: message, agent_id: agentId } }). Close bar. Catch errors and show in hint area using textContent.
     - Otherwise: create unassigned task via api('/tasks', { method: 'POST', body: { title: value } }). Close bar. Catch errors and show in hint area using textContent.
   - If key is Escape: close bar (redundant safety)

**Keyboard Shortcuts** -- replace the existing document keydown listener:

5. The current handler blocks Cmd+K because it returns early on e.metaKey. Replace the entire keydown listener with a restructured version that:
   - FIRST checks for Cmd+K / Ctrl+K: if (e.metaKey or e.ctrlKey) AND (e.key is 'k' or 'K'), preventDefault, call toggleCommandBar, return
   - THEN checks if typing in form field (INPUT/TEXTAREA/SELECT) and returns if so
   - THEN checks for other modifier keys and returns if so
   - THEN handles number key navigation: keys 1-4 map to the four tabs
   - THEN handles Escape: removes any element with class "overlay"

   This ensures Cmd+K works even when the command bar input is focused (checked before the form field guard).
