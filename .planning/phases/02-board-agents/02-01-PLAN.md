  <name>Task 1: Add HTML5 drag-and-drop to kanban board with visual feedback</name>
  <files>public/app.js, public/styles.css</files>
  <action>
Modify the board rendering and add drag-and-drop in public/app.js:

1. **Make task cards draggable.** In the `taskCard()` function, add `draggable="true"` to the `.board-card` div (it already has `data-task-id`).

2. **Add drag event wiring via event delegation on the board container.** In `renderBoard()`, after setting `container.innerHTML`, add event delegation for drag events on the container element:
   - `dragstart`: Find closest `.board-card`, set `e.dataTransfer.setData('text/plain', taskId)`, set `e.dataTransfer.effectAllowed = 'move'`, add class `dragging` to the card.
   - `dragend`: Remove `dragging` class from any card that has it. Remove `drag-over` class from all columns.

3. **Make columns into drop zones.** In the `renderBoard()` function, add `data-status` attribute to each `.board-column` div (use the status key: todo, in_progress, review, done). Then wire via delegation on container:
   - `dragover` on `.board-column`: `e.preventDefault()`, `e.dataTransfer.dropEffect = 'move'`, add class `drag-over` to the column.
   - `dragleave` on `.board-column`: Remove `drag-over` class. Use `e.relatedTarget` check -- only remove if the related target is not a child of the column (prevents flicker when dragging over child elements).
   - `drop` on `.board-column`: `e.preventDefault()`, get taskId from `e.dataTransfer.getData('text/plain')`, get target status from column's `data-status`. Remove `drag-over` from all columns.

4. **Optimistic UI update on drop.** When a card is dropped:
   - Find the task in `state.tasks` by ID
   - If the task's current status equals the target status, do nothing (same column)
   - Otherwise, create a new tasks array with the task's status changed to the target status
   - Assign to `state.tasks` (triggers Proxy set -> re-render)
   - Then fire `api('/tasks/' + taskId, { method: 'PATCH', body: { status: targetStatus } })` -- fire-and-forget with a `.catch()` that reverts the state on error

5. **Sort order within columns.** When dropping, calculate sort_order based on drop position. Use a simple approach: find the `.board-card` element closest to the drop point using `e.clientY` and the cards' bounding rects. Insert before that card's sort_order. Send `sort_order` in the PATCH body. For the initial implementation, use `Date.now()` as sort_order on drop (newer = bottom) -- this is sufficient since cards within a column are already sorted by sort_order then created_at DESC.

Add drag visual styles in public/styles.css:

```css
/* Drag-and-drop states */
.board-card.dragging {
  opacity: 0.4;
  border: 1px dashed var(--accent-green);
}

.board-column.drag-over {
  background: rgba(0, 255, 136, 0.05);
  border: 1px dashed var(--accent-green);
  border-radius: var(--radius-md);
}

.board-column.drag-over .board-column-header {
  color: var(--accent-green);
}
```

IMPORTANT: Do NOT use inline `ondragover` / `ondrop` attributes. Use event delegation on the container element, consistent with the existing pattern (container.onclick). This avoids polluting global scope and survives innerHTML re-renders.

IMPORTANT: The existing `container.onclick` handler for `data-action="new-task"` must be preserved. Extend the event delegation, don't replace it.
